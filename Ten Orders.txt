1- Pull Request :
A request to merge changes from one branch to another, typically reviewed and approved by collaborators.


2- Tag & Release:
Tag: A marker for a specific point in the repository's history, often used for versioning.
Release: An annotated version of a tag, including compiled binaries and release notes.


3- gitignore file  :
A file that specifies which files and directories Git should ignore, preventing them from being tracked.


4- SSH (Secure Shell):
 A protocol for secure remote connections, used in Git for authenticating access to repositories via SSH keys.

5-Clone : 
 Creates a local copy of a remote repository.


6-Commit:
 Records changes to the local repository with a descriptive message.

7-Branch:
 Creates a new branch, allowing for independent development.

8-Push : 
Uploads local repository changes to a remote repository.

9-pull :
Fetches updates from the remote repository and merges them into the local.

10-Status :
Displays the state of the working directory and the staging area, showing which changes have been staged, which haven't, and which files are not being tracked by Git.
